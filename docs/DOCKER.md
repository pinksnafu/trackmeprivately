# Docker Deployment

Privacy Tracker can run as a production Docker container with a persistent
SQLite database volume. The container starts the Next.js server, verifies the
required production secret, and runs `prisma db push` against the configured
SQLite database before the app starts.

## Quick Start

Create a local `.env` file for Docker Compose:

```env
NEXTAUTH_SECRET="replace-with-a-strong-random-secret"
ALLOWED_RP_ID="localhost"
ALLOWED_RP_ORIGIN="http://localhost:3000"
```

Generate a strong secret with:

```bash
openssl rand -base64 32
```

Build and start the container:

```bash
docker compose up --build
```

The app will be available at `http://localhost:3000`. On first boot, the setup
token is logged by the app and written inside the container at
`prisma/setup_token.txt`.

## Production Settings

For production, set the passkey origin values to the public analytics host:

```env
NEXTAUTH_SECRET="choose-a-strong-random-key"
ALLOWED_RP_ID="analytics.example.com"
ALLOWED_RP_ORIGIN="https://analytics.example.com"
```

The Compose file persists SQLite data in the named `tracker-data` volume and
uses:

```env
DATABASE_URL="file:/data/trackmeprivately.db"
```

Do not mount a host directory over `prisma/`; that can hide the schema required
by Prisma. Mount or persist `/data` only.

## Caddy Reverse Proxy

Point Caddy at the container port:

```caddy
analytics.example.com {
    reverse_proxy 127.0.0.1:3000
}
```

Then set:

```env
ALLOWED_RP_ID="analytics.example.com"
ALLOWED_RP_ORIGIN="https://analytics.example.com"
```

## Render Blueprint

This repository includes `render.yaml` for Render Blueprint deployments:

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/pinksnafu/trackmeprivately)

After creating the service, set `ALLOWED_RP_ID` and `ALLOWED_RP_ORIGIN` to the
Render host or your custom analytics domain. `NEXTAUTH_SECRET` is generated by
Render, and SQLite data is persisted on a `/data` disk.

## Railway

Railway can build this repository from the Dockerfile. Create a service from
the GitHub repository, attach persistent storage mounted at `/data`, and set:

```env
DATABASE_URL="file:/data/trackmeprivately.db"
NEXTAUTH_SECRET="choose-a-strong-random-key"
ALLOWED_RP_ID="your-railway-or-custom-domain"
ALLOWED_RP_ORIGIN="https://your-railway-or-custom-domain"
```

Publish a Railway template from that configured service if you want a reusable
one-click Railway deploy flow for other users.
